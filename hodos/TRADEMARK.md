# Hodos Trademark Policy

This policy explains how you may (and may not) use the Hodos name when
working with the open-source code in this repository.

It is written in plain English. If something here is unclear, ask first
(see "How to ask" below) — that is always cheaper than guessing.

## What this policy covers

The name **Hodos** is a trademark of Hussain Ahmed, used to identify
this open-source customer journey intelligence engine.

The Apache 2.0 license that covers the code in this repository **does
not** grant you any rights to use this name. (See Apache 2.0,
Section 6.) That is why this separate document exists.

## What you can do without asking

You do not need permission to:

- **Refer to the project by name.** Saying "we use Hodos", "this is
  built on Hodos", "compatible with Hodos", "powered by Hodos" is fine.
  This is *nominative use* and is what trademarks are for.
- **Distribute unmodified copies of the code** under the name Hodos.
  If the code is exactly what is in this repository, the original name
  applies.
- **Write tutorials, blog posts, talks, or books** about Hodos. Use
  the name to describe what you are talking about.
- **Use the name in academic or journalistic work.**

When you do any of the above, please:

- Spell the name as written (Hodos, capitalised).
- Make clear that your work is your work, not an official Hodos release.

## What you must NOT do without asking

You must not, without written permission:

- **Release a modified fork under a Hodos name.** A fork called
  "HodosX", "Hodos Plus", "Hodos by Acme", "AcmeHodos", "Hodos-Pro",
  or any similar variation is not allowed. Pick a different name for
  your fork — you can still say "based on Hodos" or "powered by Hodos"
  in the description.
- **Use the Hodos name as the name of a product, service, company,
  app, hosted offering, or domain.** This includes domain names,
  social handles, App Store / Play Store listings, and SaaS products.
- **Imply that your fork or service is the official Hodos** — for
  example by using project marketing copy in a way that would confuse
  a reasonable user.
- **Sublicense, transfer, or sell** the right to use the Hodos name.

## A simple test

Ask yourself: *"If someone read my product / repo / page name without
any context, would they think this is the official Hodos project?"*
If the answer is "yes" or "maybe", you need permission. If the answer
is clearly "no, it's obviously a third-party thing referring to
Hodos", you are in the *nominative use* zone above.

or research about the project's history), the same nominative-use

## Modifications and the original code

The Apache 2.0 license requires modified files to carry prominent
notices that you changed them. If your modifications are significant,
please also:

- Make clear in user-facing places (README, About page, app store
  description) that this is a modified version, not the official Hodos.
- Keep the LICENSE and NOTICE files from the original repository.

This is not a trademark requirement on its own — it is the Apache 2.0
license already asking for it. Following both keeps everyone honest.

## How to ask

For permission requests, partnership questions, or anything you are
not sure about, email **hello@cjipro.com** with the subject line
starting "Trademark:". A short email describing what you want to do
and why is fine — formal letters are not required.

Most reasonable requests get a "yes". The point of this policy is to
prevent confusion in the market, not to block legitimate use.

## Changes to this policy

This document may change as the project grows. Material changes will
be announced in the repository's release notes. Past use under a
previous version of the policy continues to be allowed under that
version's terms.

## What this policy is not

- This policy is not a trademark *registration*. Registration in
  specific jurisdictions is a separate process and may follow if
  needed.
- This policy is not legal advice. If you are unsure whether your use
  is allowed, ask before acting.
- This policy does not modify the Apache 2.0 license that covers the
  code. The code license and the trademark policy are independent.
